# 🎾 Tennis Match Simulator — запуск на GitHub Pages

Симулятор теннисных матчей (Monte Carlo) с опциональным подключением live-данных
через api-tennis.com.

## Файлы

- **index.html** — сам сайт (переименуй `tennis-sim.html` в `index.html`)
- **cors-proxy-worker.js** — серверный прокси для Cloudflare Workers (нужен для live-данных)

---

## ЧАСТЬ 1. Запустить сайт на GitHub Pages (5 минут)

1. Создай аккаунт на https://github.com (если ещё нет).
2. Нажми **New repository** (зелёная кнопка). Имя любое, например `tennis-sim`.
   Поставь галочку **Public**. Нажми **Create repository**.
3. На странице репозитория нажми **uploading an existing file**
   (или **Add file → Upload files**).
4. Переименуй `tennis-sim.html` в **`index.html`** и загрузи его.
   Нажми **Commit changes**.
5. Открой **Settings** (вкладка вверху) → слева **Pages**.
6. В разделе **Build and deployment** → **Source** выбери **Deploy from a branch**.
   Branch: **main**, папка: **/ (root)**. Нажми **Save**.
7. Подожди 1-2 минуты. Вверху страницы Pages появится ссылка вида:
   `https://ТВОЙ_ЛОГИН.github.io/tennis-sim/`
8. Открой её — сайт работает! Расчёты, ввод счёта вручную — всё доступно сразу.

На этом этапе **симулятор полностью рабочий** без всякого API — можно выбирать
игроков, покрытие, вводить live-счёт вручную и считать вероятности.

---

## ЧАСТЬ 2. Подключить live-данные (нужен прокси — иначе CORS заблокирует)

> ⚠️ api-tennis.com — серверный API, он НЕ разрешает запросы напрямую из браузера.
> Поэтому нужен прокси-посредник. Cloudflare Workers — бесплатно и быстро.

1. Зайди на https://dash.cloudflare.com → зарегистрируйся (бесплатно).
2. Слева **Workers & Pages** → **Create** → **Create Worker**.
3. Сотри весь код по умолчанию, вставь содержимое файла **cors-proxy-worker.js**.
4. Нажми **Deploy**. Получишь URL вида:
   `https://tennis-proxy.ТВОЙ_ЛОГИН.workers.dev`
5. Добавь свой API-ключ как секрет (чтобы он не светился в коде):
   - Открой свой Worker → **Settings** → **Variables and Secrets** → **Add**
   - Type: **Secret**, Name: **`API_KEY`**, Value: твой ключ от api-tennis.com
   - Нажми **Deploy** ещё раз.
6. Скопируй URL воркера. На сайте в блоке «🌐 Live-данные» вставь его
   в поле **«URL прокси»**. Поле «API ключ» можно оставить пустым —
   ключ теперь живёт на прокси.
7. Нажми **«📡 Загрузить live-матчи»** — должны появиться текущие матчи.

---

## ЧАСТЬ 3. Сделать APK (необязательно)

Когда сайт на GitHub Pages работает:

1. Зайди на https://www.pwabuilder.com
2. Вставь ссылку на свой сайт (`https://ТВОЙ_ЛОГИН.github.io/tennis-sim/`).
3. Нажми **Start** → дождись анализа → раздел **Android** → **Generate Package**.
4. Скачается APK, готовый к установке на телефон.

Манифест PWA уже встроен в `index.html`, так что PWABuilder его подхватит.

## ⚠️ Важно про безопасность ключа

- Если используешь **прокси** (Часть 2) — ключ хранится секретом на Cloudflare,
  в публичном коде сайта его нет. Это правильный способ.
- Если вставляешь ключ прямо в поле на сайте без прокси — он сохраняется только
  в твоём браузере (localStorage), но при прямом запросе всё равно упрётся в CORS.
- **Перевыпусти ключ** в кабинете api-tennis.com, если он где-то засветился.

## Заметки про live-данные

- На **бесплатном тарифе** api-tennis обычно отдаёт только счёт матча.
  Симулятор подставит игроков, сеты, геймы, кто подаёт — и пересчитает с этой точки.
- **Детальная статистика матча** (удержание подачи, спасённые брейк-пойнты из
  point-by-point) требует **платного тарифа** (Ultra/Mega). На бесплатном этот
  блок просто не появится.
- Точных процентов попадания первой/второй подачи в API нет даже на платном —
  он фиксирует, кто выиграл очко, а не как подавали.
